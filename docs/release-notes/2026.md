## **v0.10.0-beta** - _May 30, 2026_

**What's New:** ✨

- **Per-Profile Trailer Tracking** — Trailarr now tracks trailer downloads independently for each Trailer Profile applied to a media item. Previously, a single `trailer_exists` flag covered all profiles; now every `(media, profile)` pair has its own status (`Pending`, `Downloading`, `Downloaded`, `Failed`, `Skipped`, `Unmonitored`). The media page shows the individual status of each profile for a media item.
- **`for_movies` scope on Trailer Profiles** — Each profile now has an explicit scope: **Movies only** or **Series only**. This replaces the previous implicit behaviour where one profile could match both. A profile scoped to Movies is only applied to movies; a Series profile only to series. The scope is set when creating a profile and is auto-detected for existing profiles during the upgrade (see Migration Notes below).
- **Season Trailers** — Series profiles have a new opt-in setting **Download Season Videos**. When enabled, Trailarr creates one tracking entry per season (plus one for the series overall), and searches for and downloads a trailer for each season separately. Disabled by default; existing series behaviour is unchanged until you opt in.
- **Video Type selection on Trailer Profiles** — Each profile now has a **Video Type** setting that controls what kind of video is searched for and downloaded: Trailer, Teaser, Clip, Behind the Scenes, Bloopers, Featurette, or Opening Credits. Defaults to Trailer; existing profiles are unchanged.
- **Max Count on Trailer Profiles** — Each profile has a new **Max Count** setting (default: 1) that controls how many videos to download per media item. Setting it to 3, for example, will download up to 3 trailers for each matched media item. Each download is tracked as a separate status row with its own sequence number.
- **Skip a Profile for Specific Media** — Individual `(media, profile)` tracking entries can be set to **Unmonitored** by the user. Trailarr will never attempt to download for that entry and will never overwrite this choice — useful for skipping a profile on one specific title without disabling the profile globally.
- **Issues Tracking** — Trailarr now records actionable problems as *Issues*: missing trailer files, connection failures, invalid API tokens, and missing media folders. Issues are auto-resolved when the underlying condition clears (e.g. a missing file reappears, a connection recovers). Open issues are visible in the new Issues section and via the API.
- **TMDB API Key setting** — A new **TMDB API Key** field is available in General Settings (and as the `TMDB_API_KEY` environment variable). When set, Trailarr can fetch trailer metadata directly from TMDB as an additional source.
- **Activity tab** — Issues, Events, and Logs are now grouped under a single **Activity** section in the navigation sidebar. The tab shows a badge with the count of open issues and highlights in red when any urgent issue is present.

> **⚠️ Migration Notes — please read before updating:**
>
> - **Check your Trailer Profiles after updating.** During the upgrade, Trailarr automatically assigns each existing profile a `for_movies` scope based on its filters and name. Profiles where the scope cannot be determined are **automatically disabled** and must be reviewed manually — open the profile, set the correct scope (Movies or Series), and re-enable it.
> - **One profile per scope.** If you previously used a single profile to download trailers for both movies and series, you will need to create a second profile for the other type after updating.
> - **Status display has changed.** The single per-media status indicator is replaced by per-profile status rows. You may notice media items that previously showed as "Downloaded" now showing individual profile statuses — this is expected and reflects the new granular tracking.
> - **`Stop Monitoring` behaviour change.** When a profile with *Stop Monitoring* enabled successfully downloads a trailer, it now marks other pending profile rows as Skipped rather than turning off the media's Monitor flag. The Monitor flag stays on; individual profile rows show as Skipped. This is more precise but looks different from the previous behaviour.
> - **First startup after update may be slower** on large libraries while the migration creates tracking rows for every monitored media × matched profile combination. Do not interrupt the app during this time.

**Bug Fixes:** 🐛

- **Profile priority not honoured in download order** — The trailer download loop was ordering PENDING rows by the profile's database ID instead of its `Priority` setting, and in ascending order. Profiles with a higher Priority value are now processed first, matching the documented behaviour.
- **Trailer file size limit** - Increased the file size limit for detecting it as a trailer to 250 MB as some larger bitrate files could be around 200 MB.
- **Profile settings radio buttons lagging behind selection** — Clicking a radio button on the Edit Profile page (e.g. Video Format, File Format) previously waited for the backend to respond before visually updating, causing a noticeable lag. The UI now updates immediately on click and reverts if the backend call fails.

**Other Changes:** ⚡

- **Removed `WEBUI_DISABLE_AUTH` setting** — this environment variable and its corresponding toggle in General Settings have been removed. Reverse-proxy users who relied on it to skip the login page can now achieve the same result by forwarding the `X-API-KEY` header from their proxy. When Trailarr receives a valid `X-API-KEY` header on the auth endpoint, it automatically issues a session cookie so the browser proceeds without showing the login page. See [Reverse Proxy → Bypassing the Login Page](../user-guide/reverse-proxy.md#bypassing-the-login-page) for per-proxy configuration examples.

## **v0.9.2-beta** - _May 06, 2026_

    - `[YYYY]` (4 digits) — exact year match (e.g. `[2026]`)

**Bug Fixes:** 🐛

- Fixed `trailer_exists` not being corrected to `True` when trailers already exist on disk but the flag is stale `False` (e.g. after a manual file copy or an app restart). The files scan now reconciles the flag upward as well as downward.
- Fixed `trailer_exists` being incorrectly flipped to `True` mid-download when a multi-profile download is in progress. Both the new-trailer detection loop and the reconciliation step now skip the upward correction when `monitor=True`, preventing a race condition where setting the flag early would force monitoring off and cancel pending profile downloads.
- Fixed several UI typos: `Minuimum` → `Minimum`, `Maximun` → `Maximum`, `seperate` → `separate`, `atmost` → `at most`, `atleast` → `at least`, `Youtube` → `YouTube`.

**Other Changes:** ⚡

- Refactored `scan_media_folder` in the files scan task into focused helpers (`_handle_folder_gone`, `_process_trailer_changes`) so flag reconciliation logic is consolidated in one place.
- Updated [Search documentation](../user-guide/general/index.md) to document the bracket filter syntax.
- Updated Python and npm packages to latest versions.
